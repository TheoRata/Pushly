============================================
  Pushly — Quick Start Guide
============================================

PREREQUISITES
─────────────
1. Salesforce CLI installed:
     npm install @salesforce/cli --global

   Verify: sf --version

2. Dev Hub enabled in your production org:
   - Login to production as System Admin
   - Setup → Quick Find → "Dev Hub"
   - Toggle "Enable Dev Hub" ON
   - Toggle "Enable Unlocked Packages" ON

3. Make this script executable:
     chmod +x pushly.sh


FIRST-TIME SETUP (do once)
──────────────────────────
1. Authenticate to your Dev Hub (production org):
     ./pushly.sh login
     → Choose option 1 (Dev Hub)

2. Authenticate to your sandbox(es):
     ./pushly.sh login
     → Choose option 2 (Sandbox)

3. Create a project (if starting fresh):
     ./pushly.sh setup

4. Pull metadata from your sandbox:
     ./pushly.sh pull

5. Create your first package:
     ./pushly.sh package-create


DEPLOYING A FEATURE (repeat for each release)
──────────────────────────────────────────────
1. Make your changes in the sandbox (clicks or code)

2. Pull the latest metadata:
     ./pushly.sh pull

3. Build a package version:
     ./pushly.sh version-create

4. Install in staging/UAT for testing:
     ./pushly.sh install

5. After testing passes, promote the version:
     ./pushly.sh promote

6. Install in production:
     ./pushly.sh install


SHARING WITH YOUR TEAM
──────────────────────
Copy the entire pushly/ folder to:
  - A shared network drive
  - SharePoint/OneDrive
  - Any shared location your team can access

Each team member needs to:
  1. Install the Salesforce CLI on their machine
  2. Run ./pushly.sh login to authenticate
  3. Use the same project folder for pulls and version builds


COMMON COMMANDS
───────────────
./pushly.sh help            Show all commands
./pushly.sh status          Show authenticated orgs
./pushly.sh version-list    Show all package versions
./pushly.sh pull            Get metadata from an org
./pushly.sh push            Deploy metadata directly (without packaging)


TROUBLESHOOTING
───────────────
"sf: command not found"
  → Install the CLI: npm install @salesforce/cli --global

"Not authenticated to Dev Hub"
  → Run: ./pushly.sh login → Choose Dev Hub

"Apex code coverage is below 75%"
  → Write more test classes, or use version-create without coverage
    (but you won't be able to promote to production)

"Package install failed"
  → Check the error message — usually a missing dependency
  → Try: sf package install --help for options
